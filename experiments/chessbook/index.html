Introduction

Chess is a turn-based board game where all information are known at all time. Position is symmetrical with white color starting. Each piece type moves a certain way.
The king is the most important piece and cannot move to an attacked square. A game finishes when a player cannot move on its turn. At this moment, if its king is directly attacked by the opponent pieces (in check), the game is lost, (checkmate). Otherwise the game is a draw (stalemate).

Each player aims at taking pieces that could defend the king, negate king space on the board to checkmate.

Search and Evaluation

A chess game is a succession of moves up to a final position. At each turn, each player evaluate every possible moves to pick the one that has the most chances to win.
To pick a move, the player has to visualize the future board state after doing the move and anticipate the opponent response in the list of all possible moves. This can be done multiple time an is called the depth of analysis.

All these combination of moves can be represented in a tree, the root being the starting position and first nodes being the first possible moves. In a chess engine looking for possible moves deeper and deeper in the tree is called searching, choosing the best leaf is called evaluating.
In general, chess game trees have a branching factor of around 30, it is possible to compute the number of leaves for a specific depth with the formula $ leaves = branching\_factor^{depth} $. Hence at depth of 5 is 24 millions leaves while a depth of 8 is 656 billions. A standard chess game has a depth of around 80.

Minimax

During the game, every player is looking for the best possible leaf in the tree. This behavior can be modeled with a minimax algorithm. At each node, a player will try to maximize the final score and the other one will try to minimize it. It is possible to trace a path in the tree that satisfies both players to eventually arrive at a leaf and evaluate the position.
Since the game tree can be very long, most of the time a leaf will not be the final position of the game (win/draw/loss) but another, more advance state. Chess engine evaluations have to use heuristics to determine how likely is the win from this new board state. This likelihood of win is also called score and often spans from -30 to +30.

Alpha-Beta Pruning and Iterative Deepening

Minimax requires all the leaves to be evaluated but their number grows exponentially with depth. There is a way to prune some branches without reduce faithfulness and this can be done with alpha-beta pruning.
Once a first leaf has been evaluated it gives a boundary of the attainable score for each player. Since the players want to either maximize or minimize the score value, none of them would take a path that could result in a worse score than a previously found one.
A path that will give a score higher than beta would never arise because the losing player would prefer to take the path that obtain a score of beta. Conversely, current player can avoid getting a lower score than alpha by taking the path of alpha. All paths outside of the alpha-beta bounds will be avoided by each player.
In chess, the likelihood of win is reversible, a the opponent of a player that has a 80% win has 20% chances of lose (or draw). Alpha and beta bounds can then be switched when changing to the other player point of view and the score can be negated. This is called newer version of minimax is called negamax.

The first evaluated paths give alpha-beta bounds, if the best path is selected first the narrower the bound will be and the closer it will be to the actual score of the position. With a smaller bound, most of the other paths will be pruned and very few evaluations will be necessary. With these characterics emerges iterative deepening.

A depth of 80 is not attainable with time (and memory) constraints, it is necessary to select a specific depth and to avoid getting incomplete results.
Iterative deepening consists first in searching for depth 1, then 2 and so on with the specific that westart each search with the path that got the best score previously. With alpha-beta pruning, searching depth 1, 2, 3 and then 4 successively with correct path ordering takes almost the same time as finding the best path at depth 4 directly.
DIAGRAM?
With the first path as the most important one, many optimizations are possible. The search framework can be upgraded into principal variation search. This method assumes that only the first path (principal) has to be searched up to the newer depth with a full bound of alpha and beta (also called a window). Secondary paths are searched with a null window which corresponds to a bound of beta-1 beta. DETAIL PVS, Late move reductions.
